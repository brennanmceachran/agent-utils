Web Research

OpenCode tool that runs codex web research with summaries and source URLs.

Quick start

What this gives you

  • A single tool call that returns a summary, key caveats, and source URLs.
  • Preset depth controls (fast, balanced, deep).

Try it

tool: webresearch
args:
  reason: "Decide whether to adopt Next.js MCP"
  query: "Next.js MCP guide; next-devtools-mcp repo"
  quality: balanced

Installation

Install

CLI install

Uses the same host and base path as this registry.

npx shadcn@latest add https://your-site-url/agent-utils/webresearch-opencode.v1.json

Usage

How to use it well

1) Frame the decision

State what you are trying to decide or ship. Put that in the reason field so the summary stays focused. The tool runs codex web_search_request under the hood.

2) Write the queries

Use multiple queries separated by new lines or semicolons to cover edge cases:

query: "Next.js MCP guide; next-devtools-mcp repo; Next.js 16 MCP endpoint"

3) Add context and depth

Provide versions, constraints, or must-read sources:

additional_context: "We are on Next.js 16.0.0; need official docs and repo links"
quality: deep

4) Capture output with citations

Paste the summary and sources into your design notes or PR description. If sources conflict, re-run with a narrower follow-up query.

Notes

Notes

Web research is permission-gated and requires the codex CLI on your PATH. Some sites block automated fetches; if a source is blocked, use a browser or a dedicated MCP fetch server.
