Next.js MCP

Install Next.js DevTools MCP so OpenCode can query your dev server.

Quick start

What this gives you

  • Hooks OpenCode into the Next.js dev server MCP endpoint (/_next/mcp).
  • Lets you read errors, logs, and routes without leaving the terminal.
  • Adds next-devtools-mcp to opencode.json/jsonc with a repeatable merge.

Try it

  1. Install the files from this registry.
  2. Merge the MCP payload:
node .opencode/bin/merge-mcp.mjs
  1. Start your dev server and query MCP tools (get_errors, get_logs, get_routes).

Installation

Install

CLI install

Uses the same host and base path as this registry. Includes the post-install step.

npx shadcn@latest add https://your-site-url/agent-utils/nextjs-mcp-opencode.v1.json && node .opencode/bin/merge-mcp.mjs

Usage

How to use it well

1) Install the payload files

Copy the MCP payload and merge script into your repo:

  • .opencode/mcp/next-devtools.json
  • .opencode/bin/merge-mcp.mjs

2) Merge into OpenCode config

Run the merge script from your repo root:

node .opencode/bin/merge-mcp.mjs

This updates opencode.json or opencode.jsonc with the MCP server entry.

3) Start Next.js dev server

Next.js exposes MCP at /_next/mcp while the dev server runs:

bun dev

4) Use MCP diagnostics

Query tools like get_errors, get_logs, and get_routes directly from OpenCode.

Notes

Notes

Requires Next.js 16 or newer. The MCP endpoint only appears while the dev server is running.
